# Philosophy 03: Calm Editorial

## Summary

This approach treats DotCast as a quiet editorial workspace for curating a personal audiobook library. It balances media browsing with careful metadata editing and human-readable workflows.

## Core Idea

The user is not just consuming audiobooks; they are curating them. The UI should reduce noise, use clear page structure, and guide users through focused tasks like fixing metadata, importing suggestions, and arranging chapters.

## Why It Fits DotCast

DotCast has several workflows that need confidence:

- applying external metadata suggestions
- editing title, author, series, description, categories, and rating
- uploading and replacing files
- ordering many chapters
- sharing libraries

A calm editorial design makes these tasks feel deliberate and safe.

## Design Principles

- Use light surfaces, strong spacing, and clear section boundaries.
- Separate browsing from editing.
- Make destructive or broad actions explicit and confirmable.
- Prefer step-by-step flows for risky metadata changes.
- Use clear labels and helper text instead of icon-only actions.
- Keep the active task centered and reduce unrelated page noise.

## Library Page Direction

- Use refined cards or rows with clear title, author, duration, and next action.
- Hide advanced filters until needed.
- Show empty states and search results in plain language.
- Avoid sidebars that compete with the content.

## Edit Page Direction

- Split edit into sections: identity, categories, description, artwork, files, chapters.
- Make save status sticky.
- Put suggestions into a review flow with field-level apply controls.
- Collapse chapters by default and provide search/sort tools.

## Risks

- Can feel slower for power users if too step-based.
- Needs good defaults to avoid too many sections.
- Less visually distinctive than a cinematic direction unless typography and spacing are strong.

## Best Use

This is strongest if DotCast should feel trustworthy and comfortable for maintaining metadata-heavy audiobooks.
